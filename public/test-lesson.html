10 Commandments of JavaScript (Expanded)

10 tips to make working with JavaScript a whole lot easier!

Ending a line of code with a semicolon is important because it lets JavaScript know that the statement or that block of code is complete and will start a new statement on the next line. JavaScript statements are "instructions" to be "executed" by the web browser. For example a statement can be var i = 0;
Giving well defined names for stuff like variables, functions, and call backs is important because it helps you or other people viewing the code know what that line of code does or what its for. For example if you have a variable’s value that is equal to somethings score you would set it as something like var score = 10;
Sometimes code isn't that easy to read or understand even for the person who wrote it so leaving a comment above with some explanation or detail really helps out, especially when working with other people.
Another way to help yourself and other people read and understand your code is by using proper indentation and spacing. For example when inside of a function you would want to indent so you know that the code is in the function.
Scope determines the accessibility (visibility) of variables. Variables defined inside a function are not accessible (visible) from outside the function. Indenting also helps here to know where you are in your scope. A variable declared outside of a function is on the global scope. Console logging is very useful because it will log the value of whatever you enter into the parenthesis. For example if you want to know if your code is reaching a certain line you can put console.log(“It is working”) underneath that line.
Proper spelling is very important when in javascript because one little error can break all of your code. Javascript is also case sensitive so remember where you use capital letters. For example name is not the same as Name.
A JavaScript string simply stores a series of characters like "John Doe". A string can be any text inside quotes. For example “0” is a string that has the text 0 inside and 0 without the quotes has the value of 0.
Knowing when to use which brackets is very important. These brackets are (), [], and {}. [] bracket is used for arrays. {} brackets are used for objects. () are mainly used with functions to include events or parameters.
If you’re ever stuck or need help with ANYTHING remember to use google. Other helpful sites are w3schools, stackoverflow and mozilla MDN.
